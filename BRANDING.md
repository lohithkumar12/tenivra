## Domain (decision)

**Check result (Namecheap):** `tenivra.com` is **already registered** (since 2024). Only path there is **“Make offer”** (secondary market / negotiation) — often expensive and slow. Treat **.com** as *unavailable at retail* unless you budget for acquisition.

| Priority | Domain | Use |
|----------|--------|-----|
| **A — Public .com (recommended first)** | `gettenivra.com`, `usetenivra.com`, `trytenivra.com`, `tenivrahq.com` | Marketing site + founder email; still looks credible for a startup |
| **B — Core brand TLD (from search)** | `tenivra.io` (~$35/yr) or `tenivra.ai` (~$93/yr) | Strong for SaaS/AI; pair with a **.com** from row A for email/marketing if you can |
| **C — Budget / global** | `tenivra.org` (~$7.50/yr) or `tenivra.net` (~$13/yr) | Cheap; some buyers still expect `.com` for “main” site — fine for MVP + redirect later |
| **D — Acquisition** | `tenivra.com` | Only if you deliberately want to pay broker/market price |

**Action (when you buy):** Pick **one** primary URL for v1 (usually **A + optional B**: e.g. `gettenivra.com` + `tenivra.io` for app). Register the same day; add `www` → apex (or reverse). Re-check that your chosen `.com` variant is actually free before paying.

### Bootstrap — skip buying until the product is real

**Yes, you can wait.** Many founders build first on **free hosting URLs** (e.g. `*.vercel.app`, `*.netlify.app`, `*.github.io`) or a cheap dev setup, then buy a domain when you have something to show or your first paying pilot.

**Tradeoffs:**

- **Risk:** A cheap **`gettenivra.com`**-style name could be taken later by someone else — not fatal, but check again before launch.
- **Credibility:** Custom domain + email looks more serious for **cold outreach and pilots**; fine to defer if you are only building privately or talking to people who know you.
- **Handles:** Creating **`@tenivra`** (or backups) on X / GitHub is **free** and still worth doing early so no one squats them.

**Phase 0.3 if you defer:** Mark the name as **locked (Tenivra)**; domain purchase moves to **“before first public launch or first paid customer”** — whichever you choose as your rule.
